## Spacing Scale

The codebase uses Tailwind's default 4px-base scale, with **density exceptions** specific to the Linear-inspired board/drawer surfaces.

### Standard scale (use these by default)

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `gap-1` / `p-1` | Icon-to-text gap inside chips, dot+label spacing |
| sm | 8px | `gap-2` / `p-2` | Compact element spacing inside cards |
| md | 16px | `gap-4` / `p-4` | Default element spacing in panels |
| lg | 24px | `gap-6` / `p-6` | Section padding inside drawers |
| xl | 32px | `gap-8` / `p-8` | Page section gaps |
| 2xl | 48px | `gap-12` / `p-12` | Major section breaks |
| 3xl | 64px | `gap-16` / `p-16` | Page-level rhythm |

### Density exceptions (LOCKED — Linear pattern, do NOT change in Phase 2)

| Token | Value | Tailwind | Where it's used | Why |
|-------|-------|----------|-----------------|-----|
| card-px | 8px | `px-2` | Candidate card horizontal padding | 13px body + 8px padding gives Linear-density rows where 6 cards visible per column at 1440px |
| card-py | 6px | `py-1.5` | Candidate card vertical padding | 6px breaks the 4-base rule but is the ratio Linear uses for board rows; do NOT change without owner approval |
| card-gap | 4px | `gap-1` / `gap-1.5` (6px) | Inter-element gaps inside the card (avatar→text, icon→count) | Linear density requires sub-8px gaps inside the card |
| col-gap | 12px | `gap-3` | Between kanban columns | 12px is a tested Linear value (`gap-3`); do not use 16px (too airy) or 8px (cramped) |
| chip-px | 6px | `px-1.5` | Chip horizontal padding (StatusBadge, Chip primitive) | Chips at 13px text + 6px padding match Linear chip density |
| drawer-section-gap | 20px | custom (`space-y-5`) | Between drawer sections | 20px is the comfortable mid-point between 16 (cramped) and 24 (loose) for tab content |

**Exception list:** card-py (6px), card-gap (4-6px), chip-px (6px), drawer-section-gap (20px). All other spacing must be a multiple of 4. Do NOT introduce new exceptions in Phase 2.

### Touch targets

- Minimum interactive height: 28px on board (`h-7`) — Linear density. Mobile drawer triggers escalate to 40px (`h-10`).
- Drag handles: full card area is the drag handle on board (no dedicated handle); drawer close button at 32px (`h-8 w-8`).
- Card click target = full card; ≥40px on touch devices via media query escalation.

## Layout & Interaction Contracts

### Kanban board layout

- **Container**: `container-fluid` (clamp(1rem, 2vw, 1.75rem) padding-inline, `max-width: 120rem`)
- **Filters bar**: 48px height, sticky top below page header. White surface, border-bottom-default. Filter chips `gap-2`, debounced search input on right, "Limpar filtros" link far-right when filters dirty.
- **Board grid**: `grid-cols-6` (one column per stage group); `gap-3` (12px between columns). Each column min-width 240px, max-width 320px. Horizontal scroll on overflow.
- **Column header**: 36px tall. Eyebrow uppercase label (`TRIAGEM`, `CHECAGEM`...) + count badge `Chip` + "···" overflow menu trigger (column-level: customize fields, archive all, etc).
- **Column body**: vertical scroll inside column; cards stack with `gap-1.5` (6px). Drop zone hint visible only on drag-over (`bg-accent-soft` + dashed accent border).
- **Card**: 13px Inter, `px-2 py-1.5`. Border 1px default. Avatar 20px square (when shown). Hover: border-strong + bg-bg-subtle. Active drag: border-accent + shadow-ds-md + slight `transform: rotate(0.5deg)` for "lifting" feel (CSS only, no framer-motion).
- **Card SLA stripe**: 3px left border, replaces nothing, adds visual urgency. `border-l-status-amber` or `border-l-status-red`.

### Drawer (CandidateDrawer)

- **Width**: 480px desktop (`max-w-[30rem]`), full-width mobile slide-up.
- **Open animation**: 250ms cubic-bezier(0.16, 1, 0.3, 1) — matches existing `slide-up` keyframe.
- **Close**: ESC key, click-outside (overlay scrim `bg-text/15`), explicit close button (32px square, top-right).
- **Header** (`CandidateDrawerHeader`): 64px tall. Avatar 40px + name (display-sm 17/600) + subtitle (`{cargo} · {vaga}`) + close button. Below: action row with "Mover para...", "Adicionar nota", "···" overflow.
- **Tabs** (`CandidateDrawerTabs`): horizontal segmented; active tab has `border-b-2 border-accent`. Tabs: CV · Entrevistas · Fit · Histórico · Audit log (audit log only RH/admin).
- **Content** (`CandidateDrawerContent`): scrollable; tab body reuses `SectionCard` primitives. Sections separated by `space-y-5` (20px exception).
- **Footer** (sticky): "Mover para [próxima etapa]" primary CTA + "···" overflow with destructive actions (Descartar, Revogar consent, Anonimizar).

### Filtros inline (RS-09)

- Position: above kanban, full-width horizontal bar
- Components: 4 filter chip dropdowns (Vaga, Fase, Origem, Tag) + 1 search input + "Limpar filtros" link
- State: URL is source of truth (`useSearchParams`); chip activation toggles URL params; share-friendly
- Debounce: 300ms on search input only
- Inactive chips: `bg-bg-subtle` + `text-text-muted` + `border-default`
- Active chips: `bg-accent-soft` + `text-accent-text` + `border-accent/30` + tiny "×" remove icon

### Vagas encerradas colapsadas (RS-10)

- Section header below active jobs: 40px tall, `bg-bg-muted`, eyebrow uppercase "VAGAS ENCERRADAS (12)" + chevron right (collapsed) / chevron down (expanded)
- Default: collapsed
- Expand state persists per session (sessionStorage), NOT localStorage — fresh session starts collapsed
- When expanded: same job card grid as active section, but cards have `opacity-70` to visually de-emphasize

### Realtime silent re-render (D-04)

- No toast, no flash, no animation interruption when external move arrives
- Card slides via CSS `transition: transform 250ms cubic-bezier(0.16, 1, 0.3, 1)` from old column to new column
- During the 250ms slide, both columns reflect the candidate momentarily — acceptable trade-off vs. instant teleport

### Toast positions and durations (mini-decision LOCKED)

- Position: top-right (`<Toaster position="top-right" />`)
- Default duration: 4s
- Destructive duration: 8s
- Stack: max 3 visible; older toasts auto-dismiss
- Toast surfaces: `bg-surface` + `border-{tone}/30` + `shadow-popup`. NO solid color background (Linear style — colored borders only)

### Animation budget

- Card move (CSS only — `transform` + `transition`); 250ms cubic-bezier(0.16, 1, 0.3, 1) (`var(--ease-linear-out)`)
- Drawer open/close: 250ms slide-up
- Toast enter: 200ms fade-in + slide-up
- Drag-over column: 120ms `bg-accent-soft` fade-in
- Sparkbar segment hover: 100ms fill-darken (10% saturation increase)
- **NO** framer-motion in Phase 2 (verified: not in bundle). Use Tailwind animate utilities + existing keyframes.
