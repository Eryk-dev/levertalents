## Decisions Made

- **CJS rule file in ESM config:** Custom rules load via `createRequire(import.meta.url)`. Renaming to `.cjs` is the canonical workaround for `package.json: type=module` repos that author rules in CommonJS (the upstream ESLint default).
- **Allowlist expansion 12→31:** The plan's allowlist was generated by a single-line `grep "supabase\.from("` and missed multi-line `.from()` chains. The actual offender set on the codebase is 31 files. Expanding the allowlist preserves the gate's purpose (block NEW code from raw `supabase.from()`) without forcing a 31-file refactor in this plan. Each entry carries a TODO marker.
- **`useOrgUnits` not routed through `useScopedQuery`:** The structure UI is per-company by URL `:id`, not the active scope. Using a `__org__` prefix on the queryKey isolates it from scope-cache invalidation while still allowing partial-key match if needed.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] CJS rule file in ESM config**
- **Found during:** Task 07-01 (rule self-test)
- **Issue:** `node eslint-rules/no-supabase-from-outside-hooks.test.js` failed with `ReferenceError: require is not defined in ES module scope` because `package.json` has `"type": "module"`, so the `.js` extension is treated as ESM.
- **Fix:** Renamed both files to `.cjs` and load via `createRequire(import.meta.url)` from the ESM `eslint.config.js`.
- **Files modified:** eslint-rules/*.cjs, eslint.config.js
- **Verification:** `node eslint-rules/no-supabase-from-outside-hooks.test.cjs` exits 0.
- **Committed in:** ff55d7d

**2. [Rule 2 - Missing Critical] Expanded legacy allowlist 12 → 31 files**
- **Found during:** Task 07-01 (initial lint of `src/features/`/`src/app/`/`src/shared/`)
- **Issue:** The plan's 12-file allowlist was incomplete. Files like `src/pages/OneOnOnes.tsx`, `src/pages/hiring/JobOpeningDetail.tsx`, `src/pages/hiring/PublicJobOpening.tsx`, etc., contain multi-line `supabase\n  .from()` chains that the plan's grep missed. With the rule at `error`, the gate would fail in CI on first run with the original allowlist.
- **Fix:** Re-detected offenders via multiline-aware grep and expanded the allowlist to 31 files, each with a TODO marker.
- **Files modified:** eslint-rules/no-supabase-from-outside-hooks.cjs, .planning/phases/01-tenancy-backbone/deferred-items.md
- **Verification:** `npm run lint` reports zero `lever/no-supabase-from-outside-hooks` violations; new code (`src/features/`, `src/app/`, `src/shared/`) has 0 errors.
- **Committed in:** ff55d7d (Task 07-01 commit)

**3. [Rule 1 - Bug] AppRole widening broke exhaustive `Record<AppRole, _>` consumers**
- **Found during:** Task 07-03 (after widening `AppRole` to include `'liderado'`)
- **Issue:** `src/components/ViewAsBanner.tsx` (`ROLE_LABEL`) and `src/components/Sidebar.tsx` (`ROLE_LABEL`, `ROLE_HOME`) use `Record<AppRole, string>` which requires every key. Widening the type without updating these maps would have produced new tsc errors.
- **Fix:** Added `'liderado'` entries to all three maps + `VIEW_AS_ORDER`.
- **Files modified:** src/components/ViewAsBanner.tsx, src/components/Sidebar.tsx
- **Verification:** tsc baseline preserved at 42 errors (zero new errors).

**4. [Rule 1 - Bug] Plugin-query exhaustive-deps tripped on the chokepoint hook**
- **Found during:** Task 07-01 (after wiring plugin-query)
- **Issue:** `useScopedQuery` builds the queryKey from `['scope', scope.id, scope.kind, ...key]`. The plugin can't statically prove this captures all deps, so it warned "missing scope, fn".
- **Fix:** Inline `eslint-disable-next-line @tanstack/query/exhaustive-deps` with explanatory comment. The hook IS the chokepoint and intentionally derives the key from runtime scope.
- **Files modified:** src/shared/data/useScopedQuery.ts
- **Verification:** Lint clean on `src/shared/data/`.
- **Committed in:** ff55d7d (Task 07-01 commit)
