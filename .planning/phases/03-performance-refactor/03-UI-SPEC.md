## Typography

Inherits from `tailwind.config.ts` `fontSize` extensions and `index.css` `body` defaults. Phase 3 uses exactly these four roles, each with **at most two distinct weights** across the entire contract (400 for body, 600 for everything else):

| Role | Size | Weight | Line Height | Letter Spacing | Tailwind / class |
|------|------|--------|-------------|----------------|------------------|
| Body | 13px | 400 | 1.5 | -0.005em | base `<body>` default — used for form labels, list rows, dialog descriptions, 1:1 notes textarea content, climate question prompt |
| Label / metadata | 11–12px | 600 | 1.4 | normal | `text-xs`/`text-[12px]` — used for chip text, KV pair labels, "RH visível" badge, audit timestamps, "Expira em 24h" caption |
| Section heading | 17px | 600 | 1.25 | -0.015em | `text-display-sm` — used for page section titles ("Pauta", "Notas", "Action items", "Notas RH", "Transcrição (Plaud)", "Resumo (Plaud)") |
| Display / page title | 20px | 600 | 1.2 | -0.02em | `text-display-md` — used for page H1 ("Avaliações", "Clima", "1:1s", "Cadastrar pessoa"), KPI numbers in cycle summary cards |

**Weight inventory:** exactly two — `400` (body only) and `600` (every other role: Label/metadata, Section heading, Display, Eyebrow). Weight `500` is **not used** in Phase 3 surfaces.

Eyebrow (kicker) above section headings: `text-eyebrow` (10.5px / 600 / uppercase / +0.06em) — used for "CICLO ATIVO", "AVALIAÇÃO LÍDER → LIDERADO", "RH-ONLY", "ANÔNIMA". This is a metadata layer, not a 5th heading level; counts toward Label role.

Tabular numerics: `.tabular` class on every numeric column (counts of responses, days remaining in cycle, action-item due-date offsets, KPI numbers).

Body line-height: 1.5 (locked in `index.css` `body`).
Heading line-height: 1.2–1.25 per scale above.

### Accent reserved for (explicit list — never "all interactive elements")

The Linear indigo `--accent` (#5E6AD2) is used in Phase 3 ONLY for these elements:

1. **Primary CTA buttons** that initiate the main flow on each page: "Criar ciclo" (Avaliações), "Disparar pesquisa" (Clima), "Nova 1:1" (OneOnOnes), "Cadastrar pessoa" (CreateUser), "Trocar senha" (FirstLoginChangePassword)
2. **Active state** on the toggle "Lista geral / Por par" (1:1 page) — uses `--accent-soft` background + `--accent-text` foreground
3. **Active tab indicator** on the Evaluations page tabs (Ciclos abertos / Encerrados / Templates), if tabs are used
4. **Form focus ring** on inputs/textareas (already shipped via `--ring: var(--accent)`)
5. **Selected ciclo card** on the Evaluations list (left-border or background tint of `--accent-soft`)
6. **Submit success state** on EvaluationForm (subtle accent border on the saved section)
7. **The "Copiar mensagem" primary button** on the post-cadastro modal (D-20)

Accent is NOT used for:
- Generic body links (use `--text` underlined, not indigo)
- Secondary buttons (use Btn `secondary` — surface background, text foreground)
- Static badges that are not selection state (use neutral chip)
- Decorative dividers, separators, or placeholder content

## Interaction Contract

### Forms (PERF-07)

- `react-hook-form@7.73` + `@hookform/resolvers@5.2.2` + `zod@3.25` (locked — DO NOT upgrade Zod to v4, blocked by AF-13 in REQUIREMENTS.md and project CLAUDE.md).
- Zero `as any` casts in form code (lint guard).
- Optimistic submission: `onMutate` → `cancelQueries` → `setQueryData` → `onError` rollback → `onSettled` invalidate. Pattern matches Phase 2 `useMoveApplicationStage`.
- Submit button states: `idle` (default), `submitting` (spinner inside Btn, label changes to "Salvando…"), `success` (brief 600ms accent border on saved section then back to idle), `error` (toast + inline error if field-level).
- EvaluationForm Zod resolver is generated dynamically from `template_snapshot.sections[].questions[]` (D-07). Each question type maps to: `scale_1_5` → `z.number().int().min(1).max(5)`, `text` → `z.string().min(1)` (if required) / `z.string().optional()`, `choice` → `z.enum([...options])` (required) / `z.enum([...]).optional()`.

### 1:1 form layout (D-18 split)

Two-column on desktop (≥`lg` 1024px), single-column on mobile. Sticky header with: title + RH-visible badge + status chip + Save button. Sections appear in this vertical order:

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Form labels | Every input has a visible `<Label>` (shadcn Form already wires `htmlFor`/`id`) |
| Error association | RHF + `<FormMessage>` connects errors to inputs via `aria-describedby` |
| Focus ring | `--ring: var(--accent)` already configured; `focus-ring` utility class for custom controls |
| Keyboard navigation | All Radix primitives ship with keyboard support; verify Tab order on `OneOnOneMeetingForm` after the split (sections in DOM order = visual order) |
| Screen reader announcements | Toast `role="status"` for success, `role="alert"` for error (Sonner default) |
| Color-not-only | Cycle status chip pairs color with text label ("Em andamento", "Encerrado") — never color alone |
| Anonymous flow | Climate response form has `aria-label="Pesquisa de clima — anônima"` on the form root; submit button explicitly says "Enviar respostas" |
| Anti-spoof on RH note | RH-only section uses Radix `<Tabs>` or conditional render — RH role is checked via CASL `useAbility()`, NOT just CSS visibility |
